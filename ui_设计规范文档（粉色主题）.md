# 一、设计目标与整体风格

## 1.1 设计目标

本 UI 设计规范用于指导《新能源汽车动力系统与维护》理虚实一体化课程平台的**纯前端页面设计与实现**，确保：
- 视觉风格统一
- 组件可复用
- 适配 Vibecoding（AI 辅助生成 UI）开发方式
- 与职业教育场景匹配，**专业但不严肃、柔和但不幼稚**


## 1.2 整体视觉风格

- **主色调**：粉色系（低饱和、浅渐变）
- **关键词**：柔和、圆润、卡片化、轻拟物
- **设计取向**：教学友好型 UI，而非工具型系统


# 二、色彩规范（Color System）

## 2.1 主色与辅助色

| 用途 | 色值建议 | 说明 |
|----|----|----|
| 主背景色 | #FFF3F4 | 页面整体背景 |
| 主品牌粉 | #FF8FA3 | 强调色、按钮、选中态 |
| 浅粉强调 | #FFD6DD | 卡片边框 / Hover |
| 成功绿 | #35C759 | 正确 / 选择题标签 |
| 警示橙 | #FF9F0A | 连线题标签 |
| 信息蓝 | #3B82F6 | 排序题标签 |
| 正文主色 | #1F2937 | 标题 / 正文 |
| 次级文字 | #6B7280 | 说明性文本 |

> 禁止使用高饱和红色或深粉色作为大面积背景。


## 2.2 渐变使用规范

- 顶部 Tab、主容器可使用 **粉色轻渐变**：

```
linear-gradient(180deg, #FFF6F8 0%, #FFFFFF 100%)
```


# 三、字体与排版规范（Typography）

## 3.1 字体

- 中文：PingFang SC / HarmonyOS Sans
- 英文 / 数字：Inter


## 3.2 字号层级

| 场景 | 字号 | 字重 |
|----|----|----|
| 页面主标题 | 20px | Semibold |
| 卡片标题 | 16px | Medium |
| 正文内容 | 14px | Regular |
| 标签文字 | 12px | Medium |
| 辅助说明 | 12px | Regular |


# 四、整体布局规范

## 4.1 页面结构

```
顶部导航 Tab

主内容容器（白色圆角卡片）
 ├─ 内容区
 └─ 浮动实训助手
```

- 页面最大宽度：1200px
- 内容区居中
- 背景与内容区明显分层


## 4.2 卡片化设计原则

- 所有功能模块使用卡片承载
- 卡片圆角：16px
- 卡片阴影：

```
0 8px 24px rgba(255, 143, 163, 0.12)
```


# 五、顶部 Tab 设计规范（重点）

## 5.1 Tab 样式

- 整体为**胶囊型容器**
- Tab 横向排列，居中
- 非选中态：透明背景 + 深灰文字
- 选中态：白色背景 + 粉色文字 + 阴影


## 5.2 Tab 尺寸

- 高度：44px
- 单个 Tab 内边距：12px 20px
- Tab 间距：8px


## 5.3 选中态效果

- 背景色：#FFFFFF
- 字色：#FF8FA3
- 阴影：

```
0 4px 12px rgba(255, 143, 163, 0.25)
```


## 5.4 动效规范

- Tab 切换使用 200ms ease-in-out
- 禁止跳动或夸张动画



# 八、交互与状态规范

## 8.1 Hover 状态

- 卡片 Hover：
  - 阴影增强
  - 轻微上浮（2px）


## 8.2 Disabled 状态

- 不可点击卡片：
  - 透明度 0.6
  - 禁止 Hover 效果


# 九、Vibecoding UI 约束（重要）

- 所有颜色、圆角、阴影必须来自设计 Token
- 禁止在 JSX 中写死颜色值
- UI 组件必须可复用
- 单组件 JSX 不超过 200 行


# 十、适配与响应式

- 最小适配宽度：1280px
- 不强制移动端适配
- 允许缩放但不重排结构
